# Catalog Logic And Theme Correction Wave

## Исходные проблемы

- Анимации после последних волн местами воспринимались как задержка интерфейса, особенно на карточках и интерактивных элементах.
- Каталог ушёл от ожидаемой логики: первый экран содержал лишние блоки, а навигация по категориям не соответствовала сценарию "плитки -> подкатегории -> листинг".
- Тёмная тема выглядела недостаточно глубокой: база оставалась слишком графитово-фиолетовой и не везде давала нужный контраст.
- На части тёмных поверхностей вторичный и служебный текст оставался недостаточно светлым.

## Внесённые изменения

### Motion reduction

- Ослаблены motion-токены в `src/app/_assets/globals.css`.
- В storefront и admin убраны `transform`-анимации из shared transition-блоков, оставлены только лёгкие неблокирующие изменения цвета, границ и тени.
- Удалены press/hover-сдвиги для карточек и кнопок, чтобы страницы и поверхности ощущались мгновенными, а не "догружающимися".

### Theme correction

- Базовые theme-токены переведены в более глубокую почти чёрную палитру.
- Основные storefront/admin surface-градиенты затемнены, чтобы визуальная база стала ближе к чёрной.
- Акцентный цвет оставлен мягким лавандовым, но выведен из роли базового фона.

### Text readability

- Основной текст на тёмных поверхностях приведён к `var(--store-text)`.
- Вторичный текст, helper text, подписи и служебные блоки выровнены на `var(--store-text-soft)` и `var(--store-hint)` с более высоким контрастом.
- Дополнительно усилены заголовки, подписи в каталоге, карточках и admin surface-блоках.

### Catalog logic fix

- Каталог переписан под трёхшаговую структуру:
  - Экран 1: только плитки верхних разделов.
  - Экран 2: список подкатегорий выбранного раздела.
  - Экран 3: товарный листинг с поиском, сортировкой и чипами фильтров.
- Убран повторный промежуточный экран с теми же категориями в другой подаче.
- Навигация каталога теперь использует presentation-safe группировку через отдельный helper без изменения admin/import data flow.
- Сброс фильтров и возврат к подкатегориям работают явно и без тупиковых состояний.

## Остаточные ограничения / follow-ups

- Не менялись Telegram bootstrap/session/runtime, Telegram back/navigation behavior, admin guards, import flow, payment foundation и Excel import support `XLSX/XLS/XLSM/XLTX`.
- Каталогическая иерархия остаётся presentation-layer решением поверх текущих storefront категорий; если позже понадобится полноценная бизнес-иерархия, её лучше вводить отдельной задачей на уровне данных и admin UX.
